1. Third Party Requester (Client)

The third party requester in this case is an app (mobile or SPWA), that is trying to get an access token. The requester kicks off the OAuth process by posting it's authorize request to the Tyk endpoint on your gateway.

The OAuth flow is making use of the following environment variables:

  • API Listen Path: {{.APIlistenPath}}
  • Org ID: {{.OrgID}}
  • Policy ID: {{.PolicyID}}
  • API ID: {{.BaseAPIID}}
  • Gateway URL: {{.GatewayURL}}
  • Admin Secret: {{.AdminSecret}}
  • Client ID: {{.ClientID}}
  • Redirect URL: {{.RedirectURI}}
  • Response Type: {{.ResponseType}}
